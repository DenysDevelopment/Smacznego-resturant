# Дымок над горячими блюдами — дизайн

**Дата:** 2026-07-23. **Статус:** утверждён, реализован в той же сессии.

## Что

Лёгкий анимированный пар («дымок») над фото горячих блюд на карточках меню витрины.

## Решения

- **Выбор блюд — служебный тег `steam`** в существующем массиве `dishes.tags` (без миграции: колонка и её прокидка витрина/админка уже были). Владелец управляет чекбоксом «Дымок над блюдом» в форме блюда `/admin/menu/[dishId]`; чекбокс добавляет/убирает тег, свободное поле тегов его не показывает и не принимает.
- **Рендер** — `components/menu/SteamOverlay.tsx` внутри `DishCard`, только когда: есть тег `steam`, есть фото, блюдо не в стоп-листе.
- **Анимация** — чистый CSS (`app/globals.css`): три размытых белых блоба (`radial-gradient` + статичный `blur(5px)`), keyframes только `transform`/`opacity`, длительности 3.4–4.1s с разными задержками, пик opacity 0.45. Зона — над едой (`top-[22%] h-[36%]` фото-квадрата): белый пар читается на тёмной еде и тает до выхода на кремовый фон (на нём он невидим — проверено).
- **Производительность/доступность** — IntersectionObserver ставит `animation-play-state: paused` вне вьюпорта; `prefers-reduced-motion: reduce` скрывает дымок целиком.
- **Данные** — тег включён для категорий «Вареники», «Пельмени», «Котлеты и мясо» (update в конце `supabase/seed.sql` + разово в локальной БД).

## Тесты

`DishCard.test.tsx`: дымок рендерится с тегом; не рендерится без тега, без фото, в стоп-листе.
